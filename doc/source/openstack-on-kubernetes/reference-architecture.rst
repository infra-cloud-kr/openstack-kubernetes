========================
레퍼런스 아키텍처
========================

OpenStack on Kubernetes 를 멀티노드로 구성할 때의 노드 역할 분리와 최소
토폴로지, 리소스 산정 기준을 정리합니다. 실습(:doc:`../labs/index`)에 앞서 잡아
두면 좋은 설계 관점입니다.

.. note::

   기여자 작업용 골격입니다. 실습 경험과 공식 문서를 바탕으로 보강해 주세요.


노드 역할 분리
==============

* 컨트롤 플레인 노드 — OpenStack API/컨트롤러 서비스
* 컴퓨트 노드 — nova-compute(하이퍼바이저 바인딩, DaemonSet/hostPath 고려)
* 스토리지 노드 — Ceph/Cinder 백엔드
