================
운영 (Day-2)
================

배포 이후의 운영(Day-2)은 OpenStack on Kubernetes 의 진짜 가치를 결정하는
영역입니다. 업그레이드, 모니터링, 백업/복구, 스케일링을 다룹니다.

.. note::

   기여자 작업용 골격입니다. 실습 경험과 공식 문서를 바탕으로 보강해 주세요.


GitOps 기반 운영
================

* FluxCD / ArgoCD 를 통한 선언적 배포
* 구성 드리프트 감지 및 자동 동기화
* 변경의 리뷰·승인·롤백을 Git 워크플로우로 관리


업그레이드
==========

* OpenStack 릴리스 업그레이드 전략 (롤링 업그레이드)
* chart/이미지 버전 핀(pin)과 단계적 롤아웃


관측성(Observability)
=====================

* 메트릭 수집 (Prometheus)
* 로그 집계
* 알림 및 대시보드


백업 및 복구
============

* 데이터베이스(MariaDB) 백업
* 설정/시크릿 관리
* 재해 복구(DR) 시나리오
